# codex-image-api 接入指南（下游项目）

> **一句话**：codex-image-api 是一个跑在 `http://127.0.0.1:10532` 的**本地 OpenAI 兼容生图代理**，把生图请求转给你的 ChatGPT / Codex 订阅去出图——**不按张扣 OpenAI 的 API 费**（复用已付费的订阅额度，受订阅速率/额度限制）。
>
> 接入的核心只有一句：**让你的项目把 OpenAI 的 base_url 指向 `http://127.0.0.1:10532/v1`**。代码逻辑一行不用改。

---

## 0. 先判断：你的项目能不能直接接？

| 你的项目现在调的是… | 能否直接接 | 怎么做 |
|---|---|---|
| **OpenAI SDK / OpenAI 兼容 API**（`openai` python/node 包、`/v1/images/*`、`/v1/responses`） | ✅ 能 | 只改 base_url（见 §1），甚至可能零改（见 §1.A） |
| **别家厂商 API**（通义万相 / Gemini / Midjourney / Stability / Replicate / 文心一格 …） | ❌ 不能直接接 | 两条路：① 把项目改成用 OpenAI SDK；② 给 codex 加一个翻译那家 API 形状的端点（见 §6） |

判断方法：在项目里搜 `openai`、`images.generate`、`images.edit`、`responses.create`、`api.openai.com`。命中 = OpenAI 口音 = 直接接。

---

## 1. 接入（OpenAI 口音的项目）

### 1.A 最省事：什么都不用改（推荐）

本机已在 `~/.zshenv` 设了全局开关：

```bash
export OPENAI_BASE_URL=http://127.0.0.1:10532/v1
```

所以**任何用 OpenAI SDK、且没硬编码 base_url 的项目，从普通终端(zsh)启动时会自动走 codex**。新项目直接照常写生图代码即可，无需任何额外配置。

**两个前提**（不满足就要按 §1.B 显式设）：
1. **别在代码里写死 base_url**：`OpenAI(base_url="https://api.openai.com/v1")` 会盖掉全局开关。删掉那个参数即可。
2. **运行环境要能读到 `~/.zshenv`**：你手动从终端跑的都行；但 **launchd / cron / Docker / 某些 IDE 的 run 配置不读 shell profile**，要在它们自己的环境里单独设 `OPENAI_BASE_URL`（见 §1.B）。

### 1.B 显式接入（任何语言/环境通用、最稳）

不依赖全局开关，在项目里显式指定 base_url：

**环境变量法**（零改代码，推荐给 launchd/cron/Docker）：
```bash
export OPENAI_BASE_URL=http://127.0.0.1:10532/v1
export OPENAI_API_KEY=sk-local      # 占位即可：codex 不校验 key 值，但 SDK 要求非空
```

**代码法**（Python）：
```python
from openai import OpenAI
client = OpenAI(base_url="http://127.0.0.1:10532/v1", api_key="sk-local")
```

**代码法**（Node / TypeScript）：
```ts
import OpenAI from "openai";
const client = new OpenAI({ baseURL: "http://127.0.0.1:10532/v1", apiKey: "sk-local" });
```

> `api_key` 给任意非空占位串即可（如 `sk-local`）。codex 用的是 `~/.codex/auth.json` 里的订阅 token，不看你传的 key。

## 2. 三种生图调用（可直接复制）

### ① 文生图 `images.generate`

```python
r = client.images.generate(
    model="gpt-image-2",        # 会被忽略（codex 内部固定用 gpt-image-2 + gpt-5.5 编排），随便填
    prompt="a single green apple on a white plate, soft daylight",
    size="1024x1024",           # 1024x1024 / 1024x1536(竖) / 1536x1024(横) / auto
    quality="medium",           # low(测款) / medium / high(定稿) / auto
    n=1,                        # 1..8
)
import base64
open("out.png", "wb").write(base64.b64decode(r.data[0].b64_json))
```

### ② 图生图 `images.edit`（改图 / 参考图）

```python
r = client.images.edit(
    model="gpt-image-2",
    image=open("ref.png", "rb"),     # 也可传多张：[open(a,"rb"), open(b,"rb")]
    prompt="restyle in warm cinematic tone",
    size="1024x1024", quality="medium",
)
open("out.png", "wb").write(base64.b64decode(r.data[0].b64_json))
```

### ③ 导演路线 `responses.create`（gpt-5.5 自动扩写提示词）

```python
resp = client.responses.create(
    model="gpt-5.5",                       # 导演模型；codex 内部固定 gpt-5.5，填啥都行
    reasoning={"effort": "low"},           # low(快·默认) / medium / high(更真实但更慢)
    input=[{"role": "user", "content": [
        {"type": "input_text", "text": "要求皮肤白皙的网红女模特，穿驼色大衣，秋季通勤"},
        # 图生图时再加参考图（可选）：
        # {"type": "input_image", "image_url": "data:image/png;base64,<b64>"},
    ]}],
    tools=[{"type": "image_generation", "size": "1024x1536", "quality": "medium"}],
)
for o in resp.output:
    if getattr(o, "type", None) == "image_generation_call" and getattr(o, "result", None):
        open("out.png", "wb").write(base64.b64decode(o.result))   # o.result = 纯 base64
        break
```

### curl（任意语言/无 SDK 时）

```bash
curl -s http://127.0.0.1:10532/v1/images/generations \
  -H "Content-Type: application/json" \
  -d '{"prompt":"a red ceramic teacup","size":"1024x1024","quality":"low","n":1}' \
  | python3 -c 'import sys,json,base64;open("out.png","wb").write(base64.b64decode(json.load(sys.stdin)["data"][0]["b64_json"]))'
```

## 3. 参数表（generations / edits）

| 参数 | 取值 | 默认 | 说明 |
|---|---|---|---|
| `prompt` | str | 必填 | 提示词 |
| `n` | 1..8 | 1 | 出几张 |
| `size` | `1024x1024` / `1024x1536` / `1536x1024` / `auto` | `1024x1024` | 模特卡/全身用竖版 `1024x1536` |
| `quality` | `low` / `medium` / `high` / `auto` | `high` | 测款用 low、定稿用 high |
| `response_format` | `b64_json` / `url` | `b64_json` | `url` 时图存盘、返回 `http://127.0.0.1:10532/images/<name>` |
| `moderation` | `low` / `auto` | `low` | 审核档 |
| `effort`（导演相关） | `low` / `medium` / `high` | `low` | 仅影响 gpt-5.5 思考深度；high≈3-4x 时延、更真实 |
| `background` | `transparent` / `opaque` / `auto` | — | gpt-image-2 不支持 transparent |
| `output_format` | `png` / `jpeg` / `webp` | — | 上游渲染编码 |
| `reference_images` | 路径 / data-URL / `{data,mime}` 数组 | — | generations 的图生图扩展字段（≤16 张） |
| `model` | 任意 | — | **被忽略**，codex 固定 gpt-image-2 + gpt-5.5 |

## 4. 前提与边界（必读）

- **0 元 ≠ 无限**：复用 ChatGPT 订阅，受订阅的速率/额度限制；**并发上限默认 3**（`CODEX_IMAGE_CONCURRENCY`）。批量生图请串行或小并发。
- **codex 必须在线**：由 launchd job `com.taobao.codex-image` 常驻。接入前先探活：
  ```bash
  curl -s http://127.0.0.1:10532/health      # {"ok":true,"auth":"expires in ...","model":"gpt-5.5",...}
  ```
- **别硬编码 base_url**（§1.A 前提 1）。
- **非 zsh 环境单独设 env**（§1.A 前提 2）。
- **只支持图像**：codex 只有 `/v1/images/*`、`/v1/responses`、`/v1/models`、`/health`。若你把 `OPENAI_BASE_URL` 全局指过来，**非图像的 OpenAI 调用（chat/embeddings）会 404**——那种项目要么别用全局开关、要么按 §6 给 codex 加透传兜底。

## 5. 故障排查

| 现象 | 原因 / 解法 |
|---|---|
| 连不上 / Connection refused | server 没在线。`curl /health` 探活；重启：`launchctl kickstart -k gui/$(id -u)/com.taobao.codex-image` |
| `404 not found` | 调了不支持的端点（如 `/v1/chat/completions`）。codex 只支持图像端点 |
| auth 过期 / 401 | 订阅 token 过期。`codex login` 重新登录（codex CLI） |
| 出图但风格和官方略不同 | 正常：codex 用自己的 gpt-5.5 编排提示词，非官方 route B 的提示词 |
| 非图像调用报错 | 见 §4 最后一条 / §6 |

---

## 6. 别家 API 项目怎么接（进阶）

如果新项目调的是非 OpenAI 形状的 API，光换 base_url 不行。两条路：

1. **改项目代码**：改用 OpenAI SDK 调图（一次性改造）。
2. **给 codex 加翻译端点**（活放 codex、下游零改）：在 `server.py` 的 `do_POST` 加一个路由分支，写一层薄翻译——把那家 API 的请求拆成 `prompt + 参考图 + 尺寸/质量`，调内部 `generate_images(...)`，再按那家的响应形状返回。参考本仓已实现的 `_handle_responses`（OpenAI Responses API 翻译层）作模板。改完同步缓存 + `launchctl kickstart -k` 重启。

---

## 7. 回滚

删掉 `~/.zshenv` 里的 `export OPENAI_BASE_URL=...` 那行（并在显式接入处去掉 base_url），即回到官方付费 OpenAI。
